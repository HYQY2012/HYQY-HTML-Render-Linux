# 🖥️ 服务器管理工具 - 帮助中心
## 一、工具概述
### 1.1 工具简介
本工具是基于 Node.js + Web 技术开发的 macOS 风格服务器远程管理工具，支持 SSH 协议连接 Linux 服务器，集成**服务器连接管理、文件管理、终端操作、系统监控**四大核心功能，无需安装复杂客户端，开箱即用。

### 1.2 核心特性
| 功能模块       | 核心能力                                                                 |
|----------------|--------------------------------------------------------------------------|
| 服务器管理     | 一键连接、连接配置导出、多服务器信息保存                                 |
| 文件管理器     | 文件上传/下载、文本编辑、删除/重命名/移动/复制、新建文件夹               |
| 终端           | 原生 SSH 终端、快捷指令支持、复制粘贴、清屏等常用操作                   |
| 系统监控       | 实时显示 CPU 负载、内存使用率、磁盘使用率、服务器运行时间、内外网 IP 等 |
| 设备操作       | 服务器重启、关机等快捷操作                                               |

### 1.3 运行环境要求
- **前端**：Chrome/Firefox/Edge 现代浏览器（推荐 Chrome 90+）
- **后端**：Node.js 14.x 及以上版本
- **服务器**：支持 SSH 22 端口的 Linux 系统（Ubuntu/CentOS/CentOS Stream 等）
- **网络**：本地与服务器处于同一局域网，或服务器可通过公网 IP 访问

## 二、快速上手
### 2.1 启动工具
1. 安装依赖：在项目目录执行 `npm install express ssh2 ws multer`
2. 启动后端服务：执行 `node server.js`，终端显示「服务器运行在 http://localhost:3000」即为成功
3. 打开前端界面：双击 HTML 文件，进入 macOS 风格桌面

### 2.2 首次连接服务器
1. 点击桌面「服务器管理」图标，打开连接窗口
2. 填写服务器信息：
   - 服务器 IP 地址：填写服务器内网/公网 IP（如 172.29.19.65，**不要带 /20 等子网掩码**）
   - SSH 端口：默认 22，若服务器修改过端口则填写对应数值
   - 用户名：服务器 SSH 登录用户名（如 root/ubuntu，Ubuntu 系统默认禁用 root 密码登录，建议用普通用户）
   - 密码：对应用户名的 SSH 登录密码
3. 点击「连接服务器」按钮，提示「服务器连接成功」即为完成

### 2.3 导出连接配置
连接成功后，点击「导出连接配置」按钮，可将当前服务器的 IP、端口、用户名导出为 JSON 文件，方便后续快速导入使用（文件命名格式：`server-config-[IP].json`）。

## 三、功能操作指南
### 3.1 文件管理器
#### 3.1.1 基础操作
- **切换目录**：点击文件夹图标进入对应目录，点击左上角「←」按钮返回上一级
- **新建文件夹**：点击「📁 新建文件夹」按钮，输入文件夹名称后确认
- **文件上传**：点击「📤 上传」按钮，选择本地文件即可上传至当前目录
- **文件下载**：点击文件图标，自动触发下载

#### 3.1.2 高级操作
- **文本编辑**：鼠标悬停在文件上，点击「✏️」图标，打开编辑器修改内容后点击「保存」
- **重命名**：鼠标悬停在文件/文件夹上，点击「🔄」图标，输入新名称后确认
- **复制**：鼠标悬停在文件/文件夹上，点击「📋」图标，自动复制为「原名称.copy」文件
- **移动**：鼠标悬停在文件/文件夹上，点击「📦」图标，输入目标路径（如 /home/test）后确认
- **删除**：鼠标悬停在文件/文件夹上，点击「🗑️」图标，确认后删除（谨慎操作，删除后不可恢复）

### 3.2 终端
#### 3.2.1 基础使用
- 连接服务器后，点击桌面「终端」图标，自动建立 SSH 终端会话
- 支持所有 Linux 原生指令，输入后按回车执行

#### 3.2.2 快捷指令
| 快捷指令 | 功能说明       |
|----------|----------------|
| clear    | 清空终端屏幕   |
| ls       | 列出当前目录文件 |
| cd [路径]| 切换目录       |
| mkdir [名称] | 新建文件夹   |

#### 3.2.3 操作技巧
- 复制：选中终端内容后按 Ctrl+C（Windows）/ Command+C（Mac）
- 粘贴：按 Ctrl+V（Windows）/ Command+V（Mac）
- 清屏：输入 `clear` 或按 Ctrl+L

### 3.3 系统监控
- 点击桌面「系统监控」图标，自动显示服务器实时状态：
  - CPU 负载：1 分钟平均使用率（%）
  - 内存使用率：已使用内存占总内存的比例（%）
  - 磁盘使用率：/ 分区的使用比例（%）
  - 运行时间：服务器从开机到当前的时长
  - 内外网 IP：服务器的内网地址和公网出口地址（内网服务器可能无法获取公网 IP）
- 数据每 5 秒自动刷新一次，无需手动操作

### 3.4 设备操作
- **重启服务器**：在系统监控界面点击「重启服务器」按钮，确认后执行 `shutdown -r now` 指令（会中断所有服务，谨慎操作）
- **关闭服务器**：在系统监控界面点击「关闭服务器」按钮，确认后执行 `shutdown -h now` 指令（会完全关机，谨慎操作）

## 四、窗口操作
### 4.1 基础控制
- **关闭窗口**：点击窗口左上角「🔴」按钮，确认后关闭
- **最小化**：点击窗口左上角「🟡」按钮，窗口缩至桌面底部
- **最大化/还原**：点击窗口左上角「🟢」按钮，切换窗口全屏/原始大小
- **拖拽移动**：按住窗口顶部标题栏，拖动至目标位置即可

## 五、常见问题解答
### 5.1 连接类问题
#### Q1：提示「All configured authentication methods failed」
A1：原因是 SSH 身份验证失败，解决方案：
1. 检查用户名/密码是否正确（区分大小写）
2. Ubuntu 系统默认禁用 root 密码登录，建议改用普通用户（如 ubuntu）
3. 若需使用 root 登录，需修改服务器 SSH 配置：
   ```bash
   sudo vi /etc/ssh/sshd_config
   # 将 PermitRootLogin 修改为 yes
   sudo systemctl restart sshd
   ```

#### Q2：提示「connect EADDRNOTAVAIL [IP]:22」
A2：原因是填写的 IP 为广播地址（如 172.29.31.255），解决方案：
1. 登录服务器执行 `ip addr show`，找到有效 IP（如 172.29.19.65）
2. 替换前端输入框中的 IP 地址（确保最后一位不是 0 或 255）

#### Q3：提示「getaddrinfo ENOTFOUND [IP]/20」
A3：原因是 IP 地址包含子网掩码（/20），解决方案：
1. 移除 IP 后的子网掩码，仅保留纯 IP（如 172.29.19.65）
2. 手动输入 IP，避免复制粘贴带入多余字符

### 5.2 文件操作类问题
#### Q1：文件上传失败
A1：解决方案：
1. 确认当前已成功连接服务器
2. 检查服务器目标目录是否有写入权限（可通过终端执行 `chmod 777 [目录路径]` 临时开放权限）
3. 检查文件大小，避免上传过大文件（建议单文件不超过 100MB）

#### Q2：文本编辑后保存失败
A2：解决方案：
1. 确认文件路径正确，且有写入权限
2. 避免编辑系统只读文件（如 /etc/passwd，需用 root 用户登录）

### 5.3 系统监控类问题
#### Q1：外网 IP 显示「未获取」
A1：原因是服务器无外网访问权限，解决方案：
1. 登录服务器执行 `curl icanhazip.com`，测试能否访问公网 API
2. 若无法访问，检查服务器网关、DNS 配置，或联系管理员开放外网权限

#### Q2：CPU/内存/磁盘使用率显示「--」
A2：解决方案：
1. 确认服务器已安装 `top`/`free`/`df` 指令（Ubuntu/CentOS 默认自带）
2. 重新连接服务器后刷新系统监控界面

## 七、注意事项
1. 操作服务器前请确认有对应权限，避免误操作导致数据丢失
2. 禁止在生产环境中开放 root 密码登录，建议使用密钥登录（更安全）
3. 文件删除操作不可逆，删除前请确认文件无重要数据
4. 重启/关机操作会中断服务器所有服务，需提前通知相关用户
5. 工具运行期间请保持 Node.js 后端服务窗口开启，关闭后会断开所有连接

## 八、技术支持
若遇到以上文档未覆盖的问题，可通过以下方式排查：
1. 查看 Node.js 后端终端的报错信息，定位具体问题
2. 检查服务器 SSH 服务状态：`systemctl status sshd`
3. 确认服务器防火墙/安全组已开放 22 端口（入站/出站）
